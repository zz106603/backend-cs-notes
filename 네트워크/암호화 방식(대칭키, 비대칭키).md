## 대칭키 및 비대칭키 암호화 방식
- 평문을 암호화하고 복호화하는 경우 키를 사용할 수 있음
- 암복호화에 사용되는 키가 동일한 경우 **대칭키 암호화**
- 암복호화에 사용되는 키가 서로 다른 경우 **비대칭키 암호화 혹은 공개키 암호화**

---

### 대칭키 암호화
- 암호화/복호화에 같은 키를 사용
- 속도가 빠르고 데이터 암호화 효율적
- 키 교환 과정에서 탈취 위험, 통신 상대가 많을수록 키 관리 비용 증가
- AES, DES(구식), ChaCha20 등

### 비대칭키 암호화
- 공개키 (Public Key) : 누구나 알 수 있음
- 개인키 (Private Key) : 나만 알고 있는 키
- 암호화는 보통 공개키로, 복호화는 개인키로 수행
- 키 교환 문제 해결, 높은 보안성
- 대칭키보다 연산이 느림
- RSA, ECC

### 대칭키 + 비대칭키 혼합
- 비대칭키 : 세션키(대칭키)를 안전하게 교환하기 위해
- 대칭키 : 실제 데이터 암호화를 빠르게 처리하기 위해

동작 흐름 예시
1. 서버는 공개키/개인키를 가지고 있음
2. 클라이언트는 서버의 공개키로 대칭키(AES 등)를 암호화하여 전달
3. 서버는 개인키로 이를 복호화하여 동일한 대칭키 확보
4. 이후 모든 데이터는 대칭키로 빠르게 암호화하여 통신

- 키 교환은 비대칭키 -> 실제 데이터 암호화는 대칭키 -> 속도와 보안성을 동시에 확보
- 비대칭키는 TLS에서 세션키를 안전하게 공유하기 위한 초기 1회 용도
- 이후 모든 통신은 대칭키로 암호화 됨
