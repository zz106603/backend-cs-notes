## 클래스풀 IP 주소 체계
- IP 주소를 규격화된 크기별로 구분시키는 방식
- 클래스(A, B, C)별로 규격화(유형화)시켜 쉽게 식별할 수 있도록 함

```markdown
IPv4 주소는 4바이트(32비트)이며, 1바이트씩 끊어서 표기함
이때 각 바이트를 옥텟이라고 부르며, 각 옥텟은 0부터 255까지의 숫자를 표현할 수 있음
ex) 0.0.0.0 ~ 255.255.255.255
```

### A 클래스
- 1옥텟을 네트워크 주소로 사용 (초기 비트가 0(2진수)로 시작)
- 3옥텟을 호스트 주소로 사용
- 호스트 주소의 0과 255는 특수한 용도로 사용 (전자 = 네트워크 식별, 수자 = 브로드캐스트 주소)
- 2^7개의 네트워크 수와 2^24 - 2개의 호스트를 가질 수 있음

#### ▶ 시작 비트
    0xxxxxxx
#### ▶ 네트워크 호스트 구분
    1옥텟 = 네트워크
    3옥텟 = 호스트
    [네트워크][호스트][호스트][호스트]
#### ▶ 네트워크 & 호스트 수
    2^7 / 2^24 - 2 (1600만 대 이상)
---
### B 클래스
- 2옥텟을 네트워크 주소로 사용 (초기 비트가 10(2진수)으로 시작)
- 2^14개의 네트워크 수와 2^16 - 2개의 호스트를 가질 수 있음

#### ▶ 시작 비트
    10xxxxxx
#### ▶ 네트워크 호스트 구분
    2옥텟 = 네트워크
    2옥텟 = 호스트
    [네트워크][네트워크][호스트][호스트]
#### ▶ 네트워크 & 호스트 수
    2^14 / 2^16 - 2 (약 65,534대)
---
### C 클래스
- 3옥텟을 네트워크 주소로 사용 (초기 비트가 110(2진수)으로 시작)
- 2^21개의 네트워크 수와 2^8 - 2개의 호스트를 가질 수 있음

#### ▶ 시작 비트
    110xxxxx
#### ▶ 네트워크 호스트 구분
    3옥텟 = 네트워크
    1옥텟 = 호스트
    [네트워크][네트워크][네트워크][호스트]
#### ▶ 네트워크 & 호스트 수
    2^21 / 2^8 - 2 (약 254대)
---
- 클래스 풀 주소 체계는 클래스별로 네트워크 크기가 고정되어 있기 때문에, 다수의 IP 주소가 낭비될 수 있다는 한계점이 존재
- 특정 조직에 컴퓨터가 255개라면 C 클래스 주소를 사용하지 못하고, B 클래스를 사용
  - IP 주소의 낭비가 발생할 수 있음
- 이러한 문제를 해결하고, 더욱 유동적인 방식으로 네트워크를 구획할 수 있도록 클래스리스 주소 체계가 등장
---

## 클래스리스 주소 체계
- 클래스가 아닌 서브넷 마스크를 이용해 네트워크 주소와 호스트 주소를 구분하는 IP 주소 체계
- 서브넷 마스크는 네트워크 구분을 위한 비트열을 의미 (ex 192.168.10.0/23 -> 23)
- 해당 비트열에서 네트워크 주소는 연속된 1(이진수), 호스트 주소는 연속된 0(이진수)으로 표현함

#### 특정 IP 주소와 서브넷 마스크를 비트 연산을 수행하면, 네트워크 주소를 알아낼 수 있음
    서브넷 마스크가 255.255.255.0
    IP 주소가 168.168.168.168
    비트 연산을 수행하면 네트워크 주소인 168.168.168.0을 알아낼 수 있음
    이 경우, 1옥텟을 호스트 주소로 사용할 수 있음

#### 서브넷 마스크를 표현할 때는 CIDER 표기법을 사용할 수 있음
    Classless Inter Domain Routing Notation
    IP주소/서브넷 마스크 비트열의 1의 수
    - 서브넷 마스크가 255.255.255.0
    - IP 주소가 168.168.168.168
    - 168.168.168.168/24와 같이 표현할 수 있음
    - 총 32비트 중 호스트 주소 8비트를 제외한 24가 네트워크 주소

#### ex) 300대의 컴퓨터가 필요한 상황
- 필요한 호스트 = 300
- 2^9 = 512 = 충분
- 2^8 = 256 = 부족
- 즉, 호스트 비트가 9비트 필요
- 네트워크 비트 = 32 - 9 = 23
- 192.168.10.0/23
  - 11111111.11111111.11111110.00000000
  - 앞 23개의 1 = 네트워크
  - 뒤 9개의 0 = 호스트
